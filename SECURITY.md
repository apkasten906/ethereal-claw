# Security Policy

## Reporting

Do not open public GitHub issues for security-sensitive bugs.

Report security issues privately to the repository maintainers through GitHub security advisories or your internal escalation path.

## Scope

Treat the following as security-sensitive:

- Credential or token exposure
- Provider auth or permission bypass
- Unsafe file-system operations
- Arbitrary command execution risks
- Workflow steps that could spend tokens or mutate code without intended review gates
